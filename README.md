# Project PLAYBOOK AI Documentation Chatbot

This is a GitHub-ready static prototype for a Project PLAYBOOK documentation assistant.

It guides process owners through structured questions so they can provide the information needed to draft a PLAYBOOK-aligned document.

## Version

Updated using:

- `PLAYBOOK Document Matrix & QA Checklist(1).xlsx`
- `MVP Procedure for Vendor Accreditation Procedure.docx` as the output structure/template

Key alignment:

- T1 - Enterprise Process Architecture (EPA)
- T2 - Functional Policy
- T3 - Process Document
- T4 - Procedure / SOP
- T5 - Work Instruction (WI)

This version captures a **T4 Procedure / SOP** first, then generates a **T3 Process Document roll-up**.

The chatbot exports:

- `playbook-docbot-export.json` - structured answers and generated drafts
- `playbook-procedure-draft.md` - Markdown version following the MVP Procedure structure
- `playbook-process-rollup.md` - T3 Process Document roll-up
- `playbook-procedure-template-output.doc` - Word-compatible output file using the MVP Procedure structure

Important: the `.doc` export is Word-compatible HTML generated by the browser. For production, a secure backend should generate a true `.docx` using the official Word template.

## What the chatbot does

- Asks leading questions with selectable answers
- Captures PLAYBOOK-ready metadata
- Guides users on scope, triggers, end points, inputs, outputs, roles, RACI, systems, records, procedure steps, decision points, exceptions, risks, controls, evidence, KPIs, interlinking, impact assessment, approval, and review cycle
- Explains how each response affects the document
- Generates a T4 Procedure / SOP draft using the uploaded MVP Procedure template structure
- Generates a T3 Process Document roll-up in Markdown
- Exports the captured answers, procedure draft, process roll-up, and Word-compatible file
- Includes a sample Vendor Accreditation Procedure capture

## How to run locally

1. Download or clone this folder.
2. Open `index.html` directly in your browser.
3. Click **Load sample** to test the Vendor Accreditation example.
4. Click **Export Word file** or **Export package**.

No build step is required.

## How to upload to GitHub

1. Create a new GitHub repository.
2. Upload these files and folders:
   - `index.html`
   - `style.css`
   - `app.js`
   - `playbook_schema.js`
   - `word_template_output.js`
   - `templates/`
   - `README.md`
   - `.gitignore`
3. Commit the files.
4. Go to **Settings > Pages**.
5. Under **Build and deployment**, choose:
   - Source: Deploy from a branch
   - Branch: `main`
   - Folder: `/root`
6. Save.
7. Open the GitHub Pages URL once deployment completes.

## Security note

This prototype can run as a static GitHub Pages site. That is useful for demo and requirement validation, but do not place company API keys in client-side JavaScript.

For production, AI calls and true `.docx` generation should go through a secure backend such as:

- Azure Function
- Copilot Studio
- Power Platform connector
- Vercel or Netlify serverless function
- Internal O365-approved service

The browser-side API field in the demo is for local testing only.

## Recommended production direction for O365 organizations

For your organization, the practical architecture should be:

1. Front-end chatbot hosted on SharePoint, GitHub Pages, or internal web app.
2. Secure backend hosted in Azure or approved internal platform.
3. LLM service accessed using Azure OpenAI, OpenAI API, or another approved provider.
4. Backend service generates the final `.docx` using the official MVP Procedure Word template.
5. SharePoint lists/libraries used for:
   - Document Register
   - Document Links
   - Drafts
   - Published Documents
   - References & Evidence
   - Change Requests
   - Impact Assessment Tasks
6. Power Automate used for approval routing, publish workflow, read receipt, versioning, and impact assessment.
